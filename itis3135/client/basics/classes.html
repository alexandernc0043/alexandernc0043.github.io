Understanding Classes in Kotlin

Kotlin's class system is designed to simplify object-oriented programming while providing powerful features such as concise syntax, primary constructors, and data classes.

  • Class Declaration: Classes in Kotlin are declared using the class keyword followed by the class name. The primary constructor can be defined in the class header.
  • Properties and Initializers: Classes can have properties and initializer blocks (init) that run when an instance is created, allowing you to set up initial state.
  • Data Classes: Data classes automatically generate equals(), hashCode(), and toString() functions, making them ideal for storing data.
  • Inheritance: By default, classes in Kotlin are final. To allow inheritance, mark a class with the open keyword.
  • Secondary Constructors: In addition to the primary constructor, classes can have secondary constructors for additional initialization logic.

Kotlin's class features promote clarity and reduce boilerplate, enabling developers to write robust object-oriented code efficiently.

        
class Person(val name: String, var age: Int) {
    // Initializer block executes when an instance is created
    init {
        println("Person created: $name, age: $age")
    }

    // Member function to update age
    fun celebrateBirthday() {
        age++
        println("Happy birthday, $name! Now you are $age years old.")
    }
}

// Data class for storing book information
data class Book(val title: String, val author: String)

fun main() {
    // Create an instance of Person and celebrate a birthday
    val person = Person("Alice", 30)
    person.celebrateBirthday()

    // Create an instance of Book and print its details
    val book = Book("1984", "George Orwell")
    println(book)
}
